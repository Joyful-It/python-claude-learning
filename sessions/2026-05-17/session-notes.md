## 核心背诵链（按推导顺序，可顺序默写）

### 第一节：θ=w 统一（ML的底层语言）

```
θ = {w, b}  →  线性回归 ŷ = wx + b
贝叶斯 p(θ|X) 翻译成 ML：看了数据X之后，w该是多少的概率
```

| | 频率派语言 | 贝叶斯派语言 |
|---|----------|------------|
| 优化目标 | 最小化损失 | 最大化后验 p(θ\|X) |
| 数据部分 | MSE = Σ(ŷ-y)²/n | -log p(X\|θ)（假设高斯误差） |
| 信念部分 | L2/L1 正则化 | -log p(θ)（先验分布取负log） |
| 统一公式 | 总损失 = MSE + λ·罚项 | MAP = -log p(X\|θ) - log p(θ) |

**推导链：MSE = -log p(X|θ)，L2 = -log p(高斯先验)，MAP = Ridge。数学完全等价。**

---

### 第二节：频率派 vs 贝叶斯派（不是ML算法分类方式）

```
频率 vs 贝叶斯 = 统计学哲学分歧，不是 ML 算法分类

ML 算法真正分类（按计算方式）：
  wx+b派（梯度下山）：线性回归/逻辑回归/Ridge/Lasso/深度学习
  树派（贪心切蛋糕）：决策树/随机森林/XGBoost
  距离派（比邻居）：  KNN
  概率派（独立乘）：  朴素贝叶斯 4 种
  边界派（找间隔）：  SVM 4 核
```

**为什么工业 99% 用 Ridge 不用 BayesianRidge？**
算后验分布要积分/MCMC 采样，Ridge 秒出，BayesianRidge 慢百倍。除非要"预测+置信度"（医疗/金融/A/B测试）。

---

### 第三节：Ridge vs BayesianRidge

| | Ridge（频率派） | BayesianRidge（贝叶斯） |
|---|--------------|---------------------|
| 算出什么 | w = 3.2（一个数） | w 大概 2.8~3.5（分布+置信度） |
| λ 怎么来 | 手动试 / GridSearchCV | 自动从数据学 |
| 预测 | "= 42" | "= 42 ± 5" |
| 速度 | 秒级 | 慢几百倍 |
| 用在哪 | 99% 场景 | 需要置信度的场景 |

---

### 第四节：正则化的真正目的

```
目的：防过拟合，不是哲学站队

无正则化：损失 = MSE → w 迁就训练集噪音 → 测试崩
有 L2：  损失 = MSE + λw² → w 被压缩 → 训练差一点，测试好很多

正则化 = 牺牲训练精度，换泛化能力
频率派用它是防过拟合，贝叶斯派用它是表达先验 → 殊途同归
```

---

### 第五节：梯度下降 — 梯度≠最好w

```
损失函数 = 海拔地图（看错多少）
梯度     = 指南针（指方向，不说多远、不说最好在哪）
学习率   = 每一步迈多大

w_new = w_old - lr × 梯度  →  走一步，再看指南针，再走...
梯度 ≈ 0  →  到谷底 →  停下 →  这才找到最好 w

最好 w 是迭代走出来的，不是求导一步到位的。
```

---

### 第六节：决策树原理（不求导，贪心切）

```
根节点 → 试每个特征每个值切一刀 → 量"纯不纯"
→ 选最干净的一刀 → 分成左右子节点 → 继续切...

分类用 Gini/Entropy 量纯度，回归用 MSE/MAE
不求导，不用 wx+b，暴力搜索所有切分点
```

| | 决策树 | 逻辑回归 |
|---|-------|---------|
| 决策边界 | 阶梯状/轴平行 | 直线/超平面 |
| 非线性 | 天然支持 | 不支持(需人工构造) |
| 标准化 | 不需要 | 强烈建议 |
| 过拟合 | 高 | 低 |
| 稳定性 | 差(数据小变结构大变) | 稳定 |

**树模型三算法：ID3(信息增益/多叉/淘汰) → C4.5(信息增益率/连续值/剪枝) → CART(基尼/二叉树/sklearn唯一)**

### 第七节：ML 五步标准框架（学员自己总结）

```
第一步：选模型结构 → 线性/非线性/分类/回归
第二步：定义损失函数 → 回归MSE/MAE，分类交叉熵
第三步：加正则化 → L1/L2 防过拟合（= 贝叶斯先验）
第四步：优化找最好 w → 解析解 or 梯度下降（lr 控制步长）
第五步：测试集评估泛化 → 回归看MSE/R²，分类看Acc/Rec/Prec/F1
```

```python
model = Ridge(alpha=1.0)       # ①+②+③: 线性+MSE+L2
model.fit(X_train, y_train)    # ④: 梯度下降一步到位
y_pred = model.predict(X_test) # ⑤: 泛化预测
```

### 第八节：wx+b 标量 → 矩阵

```
标量：ŷ = w×x + b  ×3次 for循环
矩阵：ŷ = X @ w + b  一口气全算

∂MSE/∂w 标量 = (2/n) Σ xᵢ·(ŷᵢ-yᵢ)    逐个乘
∂MSE/∂w 矩阵 = (2/n) X.T @ (ŷ - y)    转置乘法一口气

逻辑完全一样，矩阵只是把 for 循环换成了 @ 运算。
深度学习：X @ W(100,512)@(512,64) → 加激活折弯 → 再进下一层。
```

## 今日错题本

| 问 | 错误 | 正确 |
|---|------|------|
| 神经网络要不要标准化？(复习) | 不需要，反向传播自纠正 | **需要**——特征尺度差导致梯度步长不一致 |
| 梯度求导得到什么？ | 最好 w | **方向**——最好 w 是迭代走出来的 |
| L2 和 wx+b 在一起？ | 是，L2 在 wx+b 里面 | **不是**——wx+b 是第二步(预测)，L2 是第三步(损失函数多加一项) |
| 决策树也用梯度下降？ | 应该是吧 | **不**——贪心搜索最佳切分点，不求导 |
| 工业为什么少用贝叶斯？ | 数据太多 | **计算太贵**——算分布要积分/MCMC，慢百倍 |
| 线性回归里所有模型都用损失函数求导？ | 是 | **树不用**——树用Gini/MSE衡量切的纯度，不求导 |
