你记得的那个观点其实很接近最近几年学界对于 **Scaling Law（尺度定律）** 的争论。

很多人学大模型时会形成一个误解：

```text
模型越大
↓
能力越强
↓
无限变大
↓
无限变聪明
```

实际上现在已经没人这么认为了。

我们分开讲。

------

# 一、Google《Emergent Abilities of Large Language Models》讲了什么？

2022年Google的一篇经典论文：

> Emergent Abilities of Large Language Models

作者：

- Jason Wei（后来去了OpenAI）
- Google Research

这篇论文第一次系统提出：

> 大模型存在“涌现能力”。

------

## 论文发现

他们测试了很多模型：

```text
GPT
LaMDA
PaLM
Gopher
```

规模从：

```text
100M
1B
10B
100B+
```

一路增长。

------

发现一个奇怪现象：

### 任务1：多步数学推理

小模型：

```text
参数量
↓
性能

1B      5%
10B     7%
50B     8%
```

几乎没变化。

------

结果：

```text
100B+
↓
突然
↓
60%
70%
80%
```

直接跳起来。

------

图像大概像：

```text
性能

100%
 |
 |
80%           /
 |           /
60%         /
 |         /
40%       /
 |      /
20%____/
 |
 +----------------
   参数规模
```

不是线性的。

------

## 任务2：Chain of Thought

比如：

```text
Tom有3个苹果

买了5个

送人2个

还剩多少？
```

------

小模型：

```text
直接猜
```

------

大模型：

```text
3+5=8

8-2=6

答案6
```

开始自己写推理过程。

------

作者认为：

这不是简单变强。

而是：

```text
出现了新能力
```

------

# 二、后来为什么很多人反驳这篇论文？

这是近两年特别重要的事情。

------

2023 Anthropic

2024 DeepMind

很多研究开始发现：

> 可能根本没有真正的“突然涌现”。

------

举例：

真实能力：

```text
模型A 40%

模型B 45%

模型C 50%

模型D 55%

模型E 60%
```

其实很平滑。

------

但是评测标准：

```text
必须完全正确
```

于是变成：

```text
0
0
0
0
1
```

看起来像：

```text
突然觉醒
```

------

所以现在学界有两个派别：

### 第一派

真正涌现派

认为：

```text
模型内部结构发生相变
```

出现新能力。

------

### 第二派

测量涌现派

Measurement Emergence

认为：

```text
能力一直在增长

只是评测方式
让它看起来突然出现
```

------

到今天还没完全定论。

------

# 三、GPT-2 → GPT-3 → GPT-4 的能力跃迁

这是最经典的案例。

------

# GPT-2

2019

参数：

```text
15亿
```

（1.5B）

------

当时已经震惊世界。

因为能生成文章。

例如：

```text
今天天气很好
```

继续写：

```text
阳光明媚...
```

很像人。

------

但是问题：

### 逻辑差

```text
上下文容易崩
```

------

### 数学几乎不会

```text
27+38
```

都容易错。

------

### 不会推理

```text
哥哥比弟弟高

弟弟比妹妹高

谁最高？
```

经常翻车。

------

本质：

```text
高级文本补全器
```

------

# GPT-3

2020

参数：

```text
1750亿
```

175B

一下扩大：

```text
100倍+
```

------

突然出现：

## Few-Shot

给两个例子：

```text
好电影 → Positive

烂电影 → Negative
```

然后：

```text
剧情精彩
```

GPT-3：

```text
Positive
```

------

以前做不到。

------

## 翻译

```text
中文
↓
法语
```

直接会。

------

## 代码生成

简单Python开始会写。

------

## 推理能力

开始出现。

但还不稳定。

------

很多人第一次意识到：

```text
规模扩大
真的会解锁新能力
```

------

------

## 数学


GPT-4：

```text
90%以上
```

------

## 编程


GPT-4：


------

## 多模态

```text
图片
+
文字
```

一起理解。

------

例如：

上传菜单图片。

直接：


------

## 长链推理

开始具备：

# 四、Scale Is All You Need

这是OpenAI早期的隐含信仰。

核心思想：

```text
更多参数
+
更多数据
+
更多算力
=
更强智能
```

------

GPT-2

↓

GPT-3

↓

GPT-4

一路成功。

所以大家觉得：

```text
继续扩大
就能AGI
```

------

# 五、为什么现在大家开始怀疑了？

因为2024开始碰到瓶颈。

------

## 数据不够了

互联网高质量文本：

```text
快吃完了
```

------

GPT-4训练：

已经用了：

- Common Crawl
- Wikipedia
- GitHub
- Books

等等。

------

能用的数据越来越少。

------

## 成本爆炸

GPT-3：

几百万美元

------

GPT-4：

上亿美元

------

GPT-5级别：

可能数亿美元甚至更多。

------

增长太快。

------

## 收益递减

例如：

```text
175B
↓
500B
```

提升明显。

------

但是：

```text
500B
↓
1000B
```

提升未必翻倍。

------

越来越像：

```text
花10倍钱

换5%提升
```

------

# 六、现在的新观点

目前主流已经变成：

## Scale 必须有

但是

## Scale 不够

需要：

### 更好的数据

例如：

```text
高质量推理数据
```

------

### RLHF

人类反馈强化学习

------

### RLAIF

AI反馈强化学习

------

### Test-Time Compute

推理时多思考

代表：

```text
o1
o3
DeepSeek-R1
Qwen3-Thinking
```

------

### Agent

让模型：

```text
调用工具
长期规划
自我反思
```

------

所以现在学界更认可的说法是：

```text
Scale is necessary

but not sufficient
```

即：

> 规模是必要条件，但不是充分条件。

------

如果你要做分享，我建议最后加一个非常有深度的问题：

> GPT-4 的推理能力究竟是“真正涌现”出来的，还是仅仅因为模型足够大，从而学会了更复杂的统计模式？

这个问题到 2026 年其实都没有最终答案，它正好是当前大模型研究最核心的争议之一。
