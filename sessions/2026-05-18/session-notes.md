## 核心背诵链（深度学习入门）

### 第一节：隐藏层层级抽象（核心认知）

```
第一层隐藏层 → 学「边」和「角」（离原始像素最近，只能识别最简单的几何元素）
第二层隐藏层 → 学「部件」（把边和角组合成更复杂的形状）
第三层隐藏层 → 学「完整概念」（已经能判断整张图是什么）

关键：不是设计好的！每层初始化全是随机数。
每层在梯度下降的驱动下自己"摸索"出最有用的表达方式。
越高层特征越抽象 —— 这是自然涌现，不是预设。

图像任务：第一层学边缘 → 后面学部件 → 最后学完整物体
文本任务：第一层学词义向量（king-man+woman=queen）
多模态：无论图像/语音/文本，进来都是张量，网络自动学每层该提取什么特征
```

### 第二节：一次完整训练迭代（五步）

```
① 前向传播（Forward）：用当前 w₁,w₂,w₃ 一路算到 ŷ
   x → Layer1(w₁) → Layer2(w₂) → Layer3(w₃) → ŷ

② 算损失：L = MSE(ŷ, y) 或 交叉熵(ŷ, y)

③ 反向传播（Backward）：链式法则逐层求梯度
   L → Layer3 → Layer2 → Layer1
   算出 ∂L/∂w₁, ∂L/∂w₂, ∂L/∂w₃

④ 更新（Update）：每层用自己的梯度
   w₁ = w₁ - lr × ∂L/∂w₁
   w₂ = w₂ - lr × ∂L/∂w₂
   w₃ = w₃ - lr × ∂L/∂w₃

⑤ 下一轮：用新 w₁,w₂,w₃ 从①再来一遍
```

```
PyTorch 训练循环（规范写法）：
for epoch in range(10):
    ŷ = model(X)              # ① 前向传播
    loss = loss_fn(ŷ, y)      # ② 算损失
    loss.backward()           # ③ 反向传播（自动算每层梯度）
    optimizer.step()          # ④ 更新每层 w,b
    optimizer.zero_grad()     # ⑤ 清空梯度，准备下一轮
```

**ML vs DL 唯一本质区别：**
| | ML (sklearn) | DL (PyTorch) |
|---|---|---|
| ④ 优化 | `fit()` 一步到位 | **手写 for 循环** |

### 第三节：反向传播手动计算（袖珍网络）

```
已知：x=2, w₁=3, w₂=4, y=10
网络：x → [×w₁] → z₁ → [×w₂] → ŷ
损失：L = (ŷ - y)²

① 前向：
  z₁ = w₁ × x = 3×2 = 6
  ŷ  = w₂ × z₁ = 4×6 = 24
  L  = (24-10)² = 196

② 反向：
  ∂L/∂ŷ  = 2(ŷ-y) = 2×(24-10) = 28        ← 误差信号

  ∂L/∂w₂ = ∂L/∂ŷ × ∂ŷ/∂w₂ = 28 × z₁ = 28×6 = 168
  ∂L/∂w₁ = ∂L/∂ŷ × ∂ŷ/∂z₁ × ∂z₁/∂w₁ = 28 × w₂ × x = 28×4×2 = 224

③ 更新（lr=0.001）：
  w₁ = 3 - 0.001×224 = 2.776
  w₂ = 4 - 0.001×168 = 3.832

④ 验证：用新权重再算
  z₁ = 2.776×2 = 5.552
  ŷ  = 3.832×5.552 ≈ 21.27
  L  ≈ (21.27-10)² ≈ 127  ← 比196小了！
```

**关键发现：**
```
∂L/∂w₁ = 28 × w₂ × x = 224  ← w₁的梯度比w₂的梯度大！
∂L/∂w₂ = 28 × z₁     = 168

w₂ 出现在 ∂L/∂w₁ 的计算里 → 因为误差信号必须穿过 w₂ 才能到达 w₁
越靠前的层，链越长，穿过的导数越多。这就是"反向传播"名字的由来。
```

---

### 第四节：导数为什么能给出方向？

```
∂L/∂w₁ = 224（正数）
→ w₁ 变大一点 → L 变大（损失变大=变差）
→ 应该让 w₁ 变小 → w₁ = w₁ - lr×224

∂L/∂w₁ = -224（负数）
→ w₁ 变大一点 → L 变小（损失变小=变好）
→ 应该让 w₁ 变大 → w₁ = w₁ + lr×224

导数 = 指南针：正数→往小走，负数→往大走，≈0→到谷底
```

```
损失函数 = 海拔地图（看错多少）
梯度     = 指南针（指方向，不说多远、不说最好在哪）
学习率   = 每一步迈多大

最好 w 是迭代走出来的，不是求导一步到位的。
```

### 第五节：真实网络 vs 简化版

| | 简化版（教学用） | 真实神经网络 |
|---|---|---|
| 符号 | x→z₁→ŷ | x→z₁(wx+b)→a₁(σ(z₁))→z₂(w₂a₁+b₂)→ŷ |
| b（偏置） | 省略（b=0） | 有 |
| σ（激活） | 省略（σ'=1） | sigmoid/ReLU/Tanh |
| 反向链 | L→ŷ→z₁→w₁（3步） | L→ŷ→z₂→a₁→z₁→w₁（5步） |
| ∂a₁/∂z₁ | =1（不影响） | σ'(z₁) → sigmoid会压梯度！ |

|---|---------|------|
| ML也有反向传播？ | 传统ML（线性/树/KNN/贝叶斯）有反向传播 | **没有**——只有多层网络才需要反向传播，ML用解析解或简单梯度下降，不穿多层 |

## 实操踩坑：PyTorch 泰坦尼克号数据处理

### Bug 1：SettingWithCopyWarning

```
c:\Project\python\torch_pre\10taitan.py:21: SettingWithCopyWarning:
A value is trying to be set on a copy of a slice from a DataFrame.
```

**原因**：`X = df[features]` 拿到的是视图（view），不是独立副本。往视图里填值（`X['Age'] = ...`），pandas 不确定该写回原 df 还是写副本。

**修复**：`X = df[features].copy()` —— 一刀斩断引用链。

**同类问题**：`train_test_split` 返回的 `X_train`/`X_val` 也是切片引用，标准化前同样需要 `.copy()`。

**口诀**：切片先 copy，再改不报错。

### Bug 2：TypeError: numpy.object_

```
TypeError: can't convert np.ndarray of type numpy.object_.
The only supported types are: float64, float32, float16, ...
```

**表面原因**：`X.values` 转 numpy 时得到 object 类型，不能直接转 torch tensor。

**根本原因**：`pd.get_dummies` 生成的哑变量列是 **bool** 类型。DataFrame 混着 int64、float64、bool 三类时，`.values` 统一成 object。

**验证方法**：`print(X_train.dtypes)` 看到 `Embarked_C/Q/S` 全是 `bool`。

**修复**：`X.values.astype(np.float32)` —— 显式转换后再喂 PyTorch。

| 列 | 来源 | 类型 | 问题 |
|----|------|------|------|
| Pclass, SibSp, Parch | 原始 | int64 | ✅ |
| Sex | map后 | int64 | ✅ |
| Age, Fare | 原始+标准化 | float64 | ✅ |
| Embarked_C/Q/S | get_dummies | **bool** | ❌ 混一起变 object |

**口诀**：bool 列转 float，再进 tensor。
