## 一、GPT-2 / GLM-5.2 / DeepSeek-V4 / Qwen3.7 架构对比

> 📌 Claude 速读：这份文档把四代模型从"输入→每一层干什么→输出"逐步拆解。核心演进线是——**注意力越来越省（MHA→GQA→MLA），FFN越来越稀疏（Dense→层次MoE→细粒度MoE→模态专家化），位置编码越来越长（可学习→RoPE+动态→RoPE+YaRN），最终从"能说话"走到"能自主干活(Qwen3.7 Agent环)"**。

---

### 原始 Transformer（基准）

```
输入序列 → 分词 → 词嵌入 + 正弦/余弦绝对位置编码
         ↓
┌─────────────── Encoder ×6 ───────────────┐
│ ① 多头自注意力(MHA): 每个词与所有词计算注意力     │
│ ② Add & Norm: 残差连接 + LayerNorm
│ ③ FFN: 两层全连接 + ReLU
│ ④ Add & Norm
└─────────────────────────────────────────────┘
         ↓ Encoder输出（全局上下文表示）
┌─────────────── Decoder ×6 ───────────────┐
│ ① 掩码多头自注意力(只看自己和之前的词)
│ ② Add & Norm
│ ③ 交叉注意力: Q来自Decoder, K/V来自Encoder
│ ④ Add & Norm
│ ⑤ FFN + Add & Norm
└─────────────────────────────────────────────┘
         ↓
线性层 + Softmax → 输出下一个词
```

### GPT-2 框架拆解

```
输入文本 → BPE分词 → 词嵌入 + 可学习绝对位置编码
         ↓
┌─────────────── Transformer Block ×48 ───────────────┐
│ ① LayerNorm (Pre-Norm: 先Norm再Attention)
│ ② 掩码多头自注意力(MHA + 因果掩码, 16头)
│    Q·K^T/√d_k → softmax → ×V → 多头拼接 → 线性投影
│ ③ 残差连接: x + Attention(x_normed)
│ ④ LayerNorm
│ ⑤ FFN: 升维(d→4d)→GeLU→降维(4d→d)
│ ⑥ 残差连接: x + FFN(x_normed)
└──────────────────────────────────────────────────────┘
         ↓ (重复48层)
最终LayerNorm → 线性层(共享权重) → Softmax → 下一个词
```

| 特征 | 细节 |
|------|------|
| 架构 | Decoder-Only |
| 层数 | 48层 |
| 位置编码 | 可学习绝对位置编码 |
| 归一化 | LayerNorm, Pre-Norm |
| 激活函数 | GeLU |
| 注意力 | 标准MHA + 因果掩码, 16头 |
| FFN | 稠密, 升维4倍 |

> 📌 GPT-2是Decoder-Only的"祖宗"——48层纯解码器，没有Encoder、没有交叉注意力。它的设计哲学是"把生成做到极致"。可学习位置编码的问题是：训练时只见过1024长度的位置，推理超了直接懵。

---

### GLM-5.2 框架拆解

```
输入文本 → 多语言BPE分词 → 词嵌入 + RoPE旋转位置编码(动态频率缩放, 1M上下文)
         ↓
┌─────────── GLM-5.2 Transformer Block ×N ───────────┐
│ ① RMSNorm (只除均方根, 比LayerNorm更快)
│ ② 动态稀疏分组注意力 (Dynamic Sparse GQA)
│    a. Q头数>K/V头数(GQA分组共享KV)→省KV Cache
│    b. Q,K应用RoPE→编码相对位置, 天然支持长度外推
│    c. 动态路由: 轻量网络先打分→只对Top-K相关块做完整注意力
│       → 1M上下文下注意力O(n²)→近似O(n)
│ ③ 残差连接: x + Attention(RMSNorm(x))
│ ④ RMSNorm
│ ⑤ 层次化MoE前馈网络 (Hierarchical MoE FFN)
│    a. 路由网络: 为当前token打分→选Top-K专家(如64选2)
│    b. 层次化设计: 粗粒度专家(通用语言)+细粒度专家(代码/数学/多语言)
│    c. 共享专家: 1个始终激活, 防基础信息丢失
│    d. 升维→SwiGLU激活→降维
│ ⑥ 残差连接: x + MoE_FF(RMSNorm(x))
└──────────────────────────────────────────────────────┘
         ↓
最终RMSNorm → 线性投影 → Softmax/采样 → 下一个词
```

| 特征 | 细节 |
|------|------|
| 架构 | Decoder-Only + 自回归空白填充预训练 |
| 位置编码 | RoPE + 动态频率缩放 (1M token) |
| 归一化 | RMSNorm, Pre-Norm |
| 注意力 | 动态稀疏GQA + RoPE + 动态路由 |
| FFN | 层次化MoE(粗+细粒度专家) + 共享专家 |
| 激活函数 | SwiGLU |

> 📌 GLM-5.2的杀手锏是**动态稀疏注意力**——不是傻算所有token对，而是先轻量打分再精算重点。GQA省KV Cache，动态路由省计算量，两者叠加让1M上下文跑得动。

### DeepSeek-V4 框架拆解

```
输入文本 → 分词 → 词嵌入 + RoPE(YaRN频率缩放, 超长上下文)
         ↓
┌─────────── DeepSeek-V4 Block ×N ───────────────────┐
│ ① RMSNorm
│ ② MLA (多头潜在注意力 — Multi-head Latent Attention)
│    → 极致压缩KV Cache
│    a. KV压缩到潜在空间: K,V通过下投影矩阵压到极低维
│       → 推理时只缓存压缩向量, 显存大幅降低
│    b. 上投影还原: 需要算注意力时从压缩向量还原出K,V
│    c. Q也做类似压缩-还原
│    d. RoPE解耦处理(RoPE部分单独计算)
│    e. 标准注意力: softmax(Q·K^T/√d)×V
│    → vs GPT-2: KV Cache O(n×d×heads)→O(n×d_latent)
│    → vs GLM-5.2: MLA压缩KV本身, GQA共享KV头
│ ③ 残差连接
│ ④ RMSNorm
│ ⑤ 细粒度MoE (Fine-grained MoE + Shared Expert)
│    a. 路由: 从极多专家中选极少数(如256选8)
│       → 每个专家更小, 专门化程度更高
│    b. 无辅助损失负载均衡: 动态调整路由偏置
│       → 哪个专家被用少了就降门槛, 不损失主任务性能
│    c. 共享专家: 始终激活, 处理通用模式
│    d. 升维→SwiGLU→降维→加权融合
│ ⑥ 残差连接
└──────────────────────────────────────────────────────┘
         ↓
最终RMSNorm → 线性投影 → MTP多Token预测头 → Softmax → 输出
```

| 特征 | 细节 |
|------|------|
| 架构 | Decoder-Only + MTP多Token预测 |
| 位置编码 | RoPE + YaRN频率缩放 |
| 归一化 | RMSNorm, Pre-Norm |
| 注意力 | MLA (KV潜在压缩, 极致省显存) |
| FFN | 细粒度MoE (256+选8) + 共享专家 |
| 负载均衡 | 无辅助损失, 动态偏置调整 |
| 输出头 | MTP (同时预测多个token) |

> 📌 DeepSeek-V4的MLA是独门绝技——其他家(GQA)让多个Q共享KV来省显存，MLA直接**把KV压缩到一个低维向量**里存。类比：别人是把16本笔记合并成4本(GQA)，V4是把16本笔记拍照缩成缩略图(MLA)，存得更少、用时再放大。MoE也比GLM-5.2更"细"——256个专家里挑8个，每个专家更专。

## 二、RoPE+动态缩放 vs RoPE+YaRN（位置编码扩展）

> 📌 Claude 速读：动态缩放=推理时临时调参，零成本但效果有限；YaRN=插值+分段+加温的豪华套餐，需微微训练但扩展效果稳。类比——动态缩放是手机"数字变焦"（临时放大但画质下降），YaRN是"光学变焦"（换镜头，清晰不失真）。

### 核心定位

- **RoPE+动态缩放**：底层推理策略，动态调RoPE基数，**无需训练**——轻量级"急救包"
- **RoPE+YaRN**：完整上下文窗口扩展方法，NTK-aware插值+分段+温度，通常需**少量微调**——高级"套餐"

### 机制区别

**动态缩放（Dynamic NTK）**：根据当前序列长度动态计算更大的基数。类比——"伸缩镜头"，场景不同自动调焦距

**YaRN**：三步组合拳
1. NTK-aware插值：**高频少缩放、低频多缩放**（线性插值一刀切把所有位置信息都压缩了，高频信息丢失严重）
2. NTK-by-parts：不同维度段（高频/中频/低频）用不同插值策略
3. 温度调整：softmax前加温度参数微调注意力分布

### 效果对比

| | RoPE+动态缩放 | RoPE+YaRN |
|------|------|------|
| 需训练 | ❌ 不需 | ✅ 需微调（但量少） |
| 扩展效果 | 有限，长文本降性能 | 显著稳定，可达128k+ |
| 计算开销 | 推理时近乎零 | 微调有成本，推理时零 |
| 适用场景 | 偶尔长文本/快速部署 | 稳定处理超长文档/代码库 |

### 谁用哪个

| 方案 | 代表模型 |
|------|---------|
| RoPE+动态缩放 | GLM-5.2, Qwen3.7 |
| RoPE+YaRN | DeepSeek-V4（YaRN缩放, 支持超长上下文） |

> 📌 面试一句话："动态缩放是零成本应急方案，YaRN是花少量代价一次到位。DeepSeek选了后者，GLM和Qwen选了前者——没有谁对谁错，看业务场景。"

## 三、Standard MoE → Fine-grained MoE → Hierarchical MoE 演化

> 📌 Claude 速读：MoE 两条演化路线。DeepSeek走的是"把专家拆小+加共享专家"（Fine-grained+Shared Expert），GLM走的是"专家分层，先选组再选人"（Hierarchical）。核心矛盾不同——DeepSeek解决"专家太粗，知识共享不足"，Hierarchical解决"专家太多，路由计算太贵"。

### 标准 MoE 基础

```
原来: Token → FFN → Output
MoE:  Token → Router → Expert1/Expert2/.../ExpertN → Output
例: 64个专家, Top-2, Router决定token去专家17和42
```

### Fine-grained MoE（DeepSeek 路线）

**核心**：64个大专家 → 256个小专家（甚至512个），总参数不变

```
传统: 64 × 100M参数的大专家
细粒度: 256 × 25M参数的小专家
Router可以更精细地组合: token A→专家12, token B→专家145, token C→专家231
```

**类比**：从"大部门"变成"专业小组"

### Shared Expert（DeepSeek-V2 关键创新）

```
Output = Shared Expert + Routed Experts(选中的)
```

**为什么加共享专家**：语法、基础推理、常见词汇——这些通用知识不该每次重新路由。共享专家固定激活，稳定保存通用知识，路由专家专门负责数学/代码/翻译/法律等特定领域。

### Hierarchical MoE（GLM-5.2 路线）

**核心**：先选组、再选专家——两层路由

```
Token → Router1 → 选Group(如"数学专家组")
                  → Router2 → 在组内选Expert(如专家3+专家8)
```

**类比**：先选学院，再选导师。不是从全校5000导师直接选。

**为什么用**：当专家太多（如1024个），Token与全部专家打分开销巨大。分层后：先选8个组之一→再在128个专家里选Top2，计算量大幅减少。

### 为什么 DeepSeek 不选 Hierarchical

后来发现：**GPU 通信和负载均衡才是真正瓶颈，不是 Router 打分**。所以 DeepSeek 方向变成：更多小专家 + 共享专家 + 高效路由。

### 一句话区分

| 路线 | 解决什么 | 怎么解决 | 代表 |
|------|---------|---------|------|
| Fine-grained MoE + Shared Expert | 专家太粗、知识共享不足 | 大专家→很多小专家+共享专家 | DeepSeek-V2/V3 |
| Hierarchical MoE | 专家太多、路由计算太贵 | 先选组再选专家 | GLM-5.2 |

> 📌 面试一句话："DeepSeek把专家拆细加共享，GLM把专家分层减路由。两条路线解决的是不同阶段的问题。"

## 四、位置编码演化：Sin/Cos PE → GPT-2 可学习 → RoPE

> 📌 Claude 速读：位置编码经历了三次跃迁——原始Transformer把sin/cos向量**加到输入**、GPT-2改成**可训练的**位置表但长度固定、RoPE把sin/cos**直接作用在Q/K上**让Attention天然感知相对距离。RoPE本质还是sin/cos一家人，只是用在了更合适的地方。

### 本质区别

```
原始PE = 把位置信息加到词向量里（Token+Position → 输入Embedding）
RoPE   = 把位置信息加到Attention计算过程中（旋转Q/K → 内积自带相对位置）
```

### 原始 Sin/Cos 位置编码

对每个位置 pos 和维度 2i/2i+1：
```
PE(pos,2i) = sin(pos / 10000^(2i/d))
PE(pos,2i+1) = cos(pos / 10000^(2i/d))
```

然后 `x = token_embedding + PE`。模型知道"猫在第1个位置"而不是"猫这个词本身"。

**为什么同时用 sin 和 cos**：`sin(a+b) = sin a·cos b + cos a·sin b`——相邻位置存在线性关系，模型容易学到"当前位置→下一个位置"的规律。

**为什么不同维度用不同频率**：低维度=短距离信息（变化快），高维度=长距离信息（变化慢）→一块编码同时含近距和远距信息。

### GPT-2 可学习位置编码

```
position_embedding = nn.Embedding(max_len, hidden_size)  # 直接训练位置表
x = token_embedding + position_embedding
```

**为什么后来不流行了**：训练时最大长度1024→位置表只有0~1023。推理长度2048→位置1024没有Embedding→直接废。Sin/Cos则位置10000照样能算，理论无限长。

### RoPE：Sin/Cos 的"升级版"

RoPE 不加到 Embedding 上，而是作用在 Q/K 上：

```
普通Attention: Attention = Q·K^T
RoPE:          Attention = (R_m·Q)·(R_n·K)^T
               R_m = 位置m对应的旋转矩阵
```

**旋转是什么**：Q的一部分向量在位置上做旋转。位置0→指向右，位置1→旋转一个角度↗，位置2→再转→↑...就像钟表指针。

**为什么这样就有相对位置**（最精妙的地方）：
```
(R_m·Q)^T·(R_n·K) 展开后 = Q^T·R_(m-n)·K
```
绝对位置m和n消失了，只剩m-n。位置10和11→11-10=1，位置100和101→101-100=1——**距离1的结果完全一致**。模型天然感知相对距离。

**长文本为什么更强**：原始PE训练长度2048→推理8192崩。RoPE位置只是旋转角度，2048/4096/8192/16384都能继续旋转→**外推能力更强**。

### RoPE 本质上还是 Sin/Cos 一家人

```
Transformer PE: Sin/Cos 构造向量 → 加到输入
GPT-2 PE:      学习的向量 → 加到输入
RoPE:          Sin/Cos 构造旋转角度 → 旋转Q和K
```

RoPE 是把原始 Sin/Cos 思想发挥到了更适合 Attention 的形式，不是彻底抛弃。

### 工程演化线

```
Transformer(2017): Token + Sin/Cos Absolute PE
    ↓
Transformer-XL: Relative Position
    ↓
T5: Relative Bias
    ↓
GPT-NeoX/LLaMA/Qwen/DeepSeek: RoPE
    ↓
Qwen3/Llama4/DeepSeek-V3: RoPE + 长上下文扩展(YaRN/NTK)
```

> 📌 一句话面试版："原始PE是把位置贴标签在词上让模型自己猜距离，RoPE是把位置直接编码进Q/K的几何关系里让Attention天然感知相对距离——所以长文本外推更强、参数量为0、已基本取代绝对位置编码。"

## 五、低秩（Low-Rank）——LoRA 的数学灵魂

> 📌 这是学员追问的核心概念。"低秩"三字回答了：为什么 LoRA 能用两个小矩阵代替一个大矩阵？

### 秩是什么

**秩 = 矩阵中有效独立信息的数量。** 10000 个格子但秩=8 → 本质只有 8 个独立方向，其余全是冗余。

> 类比：公司1000员工，真正出主意的核心只有8个人——其余工作都是这8人想法的延伸。

### LoRA 怎么利用

微调时 ΔW 是 1024×1024 大矩阵，但实际低秩（r=8）。分解：
```
ΔW = B × A      A: 8×1024,  B: 1024×8
```
两个"秩最多8"的小矩阵相乘 = 大矩阵里最核心的 8 维变化。其余冗余不用管。

### 为什么省力

- 满秩（r=1024）：重装整个房子，每面墙都敲 → 成本巨大
- 低秩（r=8）：只调 8 根承重柱 → 柱子一动全房风格变 → 成本极低

> **一句话**：低秩=用极少独立基向量逼近庞大但变化单一的矩阵。r=8 意味着无论 A/B 多大，真正起作用的自由变量只有 8 个——这是 LoRA 轻量高效的底层数学原理。

## 六、RoPE 上下文长度扩展——为什么微调时要动位置编码（学员独立总结）

> 📌 学员自己讲通了 LLaMA Factory 里 `rope_scaling` 选项的底层逻辑。

### 问题本质

预训练模型在固定长度（如 4096 token）上训练。RoPE 只熟悉 0~4096 的位置序号。拿 10 万字的书去微调，模型遇到位置 50000 时，RoPE 的旋转角度从未见过 → 模型全懵。

### 核心思路

**把没见过的长位置编号压缩回模型熟悉的短范围。** 0~50000 映射回 0~4096 范围内。所有 `rope_scaling` 策略的本质都是做这个"插值（Interpolation）"。

```
模型出生时只见过 0~4096 的位置
     ↓
你要它处理 50000 的位置
     ↓
方案：把 50000 压缩映射回 4096
     ↓
关键选择：用什么数学公式做这个"压缩映射"？
→ linear（线性）/ dynamic NTK（动态频率缩放）/ YaRN（分段+温度）
```

> 📌 一句话面试版："RoPE 扩展的本质是位置编号的压缩映射——把超长位置编号压缩回模型训练时熟悉的短范围，让模型'以为'自己还在处理短文本。不同策略就是不同的压缩公式。"
