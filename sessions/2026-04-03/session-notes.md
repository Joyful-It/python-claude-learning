## 今日重点内容

### 1. 广播机制深入理解

| 概念 | 掌握 |
|-----|------|
| 广播核心规则：从右对齐，不够补1 | ✅ |
| 一维 vs 二维的本质区别 | ✅ |
| 一维：[1,2,3] shape=(3,) | ✅ |
| 二维：[[1,2,3]] shape=(1,3) | ✅ |
| 一维转置不变，二维转置行列互换 | ✅ |
| 索引区别：一维用arr[0]，二维用arr[0,0] | ✅ |

**关键理解**：
- 形状是Python计算广播的依据，不是看外表
- 一维只有"行"，二维有"行"也有"列"

### 2. 梯度下降复习

**梯度下降下降的是什么？**
- 下降的是**损失值（错误程度）**
- 目标：把错误降到最低（接近0）
- 过程：猜 → 算错误 → 调参数 → 重复

### 3. 大模型完整认知

#### GPU显卡知识
| 型号 | 定位 | 说明 |
|-----|------|------|
| H100 | 最强专业卡 | 封禁，买不到 |
| A100 | 专业训练卡 | 封禁 |
| H800 | H100阉割版 | 中国能用，被限速 |
| A800 | A100阉割版 | 中国能用 |
| RTX 5090/4090/3090 | 民用游戏卡 | 性价比选择 |

**封禁原因**：限制GPU之间通信速度，不让多卡并行训练

#### PyTorch能用吗？
- CPU模式：✅ 能用，学语法够
- Intel Arc显卡：⚠️ 麻烦，效率低
- NVIDIA显卡：✅ 原生支持，推荐

#### 云服务器 vs 算力服务器
| 类型 | 作用 |
|-----|------|
| 云服务器 | 部署代码/API |
| 算力服务器（GPU） | 跑大模型 |

#### 学习路线
1. 现在：Python编程 → API调用 → 远程服务器跑模型
2. 以后：租云算力（AutoDL ~2元/小时）
3. 工作：云服务器 + 算力服务器
