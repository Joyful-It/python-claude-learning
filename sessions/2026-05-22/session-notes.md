## 第一节：权重共享——跨时刻 vs 跨门（易混淆点纠正！）

### 两个维度的"共享"

```
维度一：跨时刻共享（✅ 对的）
  W_f 在 t=1 用、t=2 用、t=3 用……同一套 W_f
  W_i 在 t=1 用、t=2 用、t=3 用……同一套 W_i
  W_o 在 t=1 用、t=2 用、t=3 用……同一套 W_o
  W_c 在 t=1 用、t=2 用、t=3 用……同一套 W_c

维度二：跨门独立（❌ 容易误以为共享）
  W_f ≠ W_i ≠ W_o ≠ W_c   — 四个门，四套参数，各学各的
```

### 类比：四个部门经理

```
遗忘门 = 裁员部经理，手册 W_f → 决定谁走
输入门 = 招聘部经理，手册 W_i → 决定谁进
输出门 = 公关部经理，手册 W_o → 对外说什么
候选   = 培训部，手册 W_c   → 生产新人才

每个经理每天都翻同一本手册（跨时刻共享）
但四个经理的手册完全不同（跨门不共享）
训练 = 业绩反馈后各自修改自己的手册
```

### 口诀

> 同门跨时刻一套参数，异门同时刻各干各的。

## 第二节：五道题核心问答（背！）

### 题1：RNN 为什么用 tanh 不用 sigmoid？（20分）

| 维度 | tanh | sigmoid |
|------|------|---------|
| 输出范围 | -1 ~ 1 | 0 ~ 1 |
| 中心 | 零中心（有正有负） | 正中心（全正） |
| 梯度范围 | (0, 1] | (0, 0.25] |
| 循环后果 | 梯度方向自由 | 梯度全同号 → 参数同涨同跌 → 收敛慢 |

**在 LSTM/GRU 中的分工**：sigmoid 做门控（0~1 天生阀门），tanh 专职产生候选信息（-1~1 内容丰富）。

### 题2：BPTT 梯度消失/爆炸的原因 + 解决方案（20分）

```
根本原因：链式连乘
  梯度 = tanh'(t) × tanh'(t-1) × … × tanh'(1) × W_h

  |W_h × tanh'| < 1 → 指数衰减 → 梯度消失
  |W_h × tanh'| > 1 → 指数爆炸 → 梯度爆炸（NaN）
```

**解决方案（按优先级）：**
```
① 架构改进：LSTM/GRU（最根本的）
② 梯度裁剪：clip_grad_norm_（max_norm=1）
③ 截断 BPTT：只回传固定步数
④ 自适应优化器：Adam
⑤ 正交初始化：让初始 W_h 谱半径接近 1
```

### 题3：LSTM 如何根治梯度消失？（20分）

**关键：细胞状态 C_t 的加法更新**

```
C_t = f_t × C_{t-1} + i_t × ～C_t
       ↑ 遗忘            ↑ 加法写入

∂C_t / ∂C_{t-1} = f_t  ← 没经过 tanh！！

如果 f_t ≈ 1：梯度几乎无损传回 → 梯度高速公路
```

普通 RNN 每次全量覆写 → 反向必穿 tanh' → 衰减。
LSTM 把更新从"覆写"改成了"加法"——加法求导系数恒为 1。

### 题4：GRU 简化了什么？（20分）

| | LSTM | GRU |
|------|------|------|
| 状态 | 2（C_t + h_t） | 1（只有 h_t） |
| 门 | 3（忘/入/出） | 2（重置 r + 更新 z） |
| 参数 | 4×RNN | 3×RNN（少 25%~30%） |

```
更新门 z 干了 LSTM 忘门+输入门两个门的活：
  h_t = (1-z) × h_{t-1} + z × 新候选
        ↑ 保留旧比例   ↑ 加入新比例
```

### 题5：注意力机制计算流程 + 解决什么问题？（20分）

**三步：**
```
① 算相似度：score(q, k_i) = q · k_i
② 归一化：  α_i = softmax(scores)     → 权重和=1
③ 加权求和：output = Σ α_i × v_i       → 挑着看
```

**解决的信息瓶颈**：传统 Seq2Seq 所有信息挤进一个 context 向量，开头被后面的覆盖。注意力让解码器每步**直接看向编码器所有时刻**，自己打分挑着看——长句也能精准取开头信息。

## 第三节：课件补充——我漏掉的三块（必背！）

### 一、双向 RNN（BiRNN）

```
结构：两个独立 RNN
  正向：h→_t = RNN(x_t, h→_{t-1})   从左往右读
  反向：h←_t = RNN(x_t, h←_{t+1})   从右往左读
  输出：h_t = [h→_t ; h←_t]         ← 拼接！

适用场景：
  命名实体识别 → 需要看前后文判断"苹果"是水果还是公司
  情感分析     → "不 太 好" vs "太 好 了"，"不"在"好"前面
  机器翻译     → 翻译一个词需要看整句
```

### 二、RNN vs Transformer 对比

| | RNN | Transformer |
|------|------|------|
| 复杂度 | O(n) 线性 | O(n²) 自注意力 |
| 并行 | ❌ 串行 | ✅ 全并行 |
| 长程依赖 | 梯度衰减 | 直接连接（Attention） |
| 流式处理 | ✅ 天然支持 | ❌ 需要技巧 |
| 控制任务 | ✅ 状态连续性好 | 弱 |
| 表达力 | 单头 | 多头注意力 |

**实践中常混合使用**：语音识别底层用 CNN/Transformer 提特征，高层用 RNN 序列建模。

### 三、工程实践 6 条（必背！）

```
① 数据预处理
   构建词表 → 序列填充(pad) → 设置 <pad> 和 <unk> 索引

② 标准模型结构
   Embedding → RNN/LSTM/GRU → Dropout → 全连接(FC)

③ ⚠️ 双向时 FC 输入维度
   hidden_size × 2  ← 因为拼接了正反向！

④ ⚠️ 必须梯度裁剪
   torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1)

⑤ 优化器配置
   Adam + 学习率调度 + Dropout(0.1~0.5)

⑥ 内存不够三招
   减 batch_size / 梯度累积 / 混合精度训练(amp)
```

## 第五节：CLIP 多模态模型（课堂新学）

### CLIP 是什么？
**双塔架构 + 对比学习**，打通图像和文本——让 AI 用自然语言理解视觉。

```
形象比喻：
  GNN  = 懂关系的专家
  Transformer = 有全局视野的大脑
  CLIP = 能看图也能读字的通才
```

### 核心架构：双塔

```
🖼️ 图像编码器（ViT 或 ResNet）    📝 文本编码器（Transformer）
        ↓                                    ↓
   视觉特征向量                          语义特征向量
        ↓                                    ↓
        →→→→→→  同一个数学空间里比大小  ←←←←←←
```

### 训练方式：对比学习 + 4亿数据

```
数据：互联网自动抓取 4 亿"图片-文本"对（无需人工标注！）
目标：匹配的图文对靠近，不匹配的远离
损失：InfoNCE → 最大化对角线（正样本），最小化非对角线（负样本）

用"数量"对抗"噪声"——
  数据里有广告/文不对题？没关系，4 亿对中正确配对在统计上占主流
```

### 杀手锏：零样本推理（Zero-Shot）

**无需重新训练，文字描述就能识别新类别。**

深层原理——"乐高式组合"：
```
CLIP 没见过"三只眼绿色外星人"整体，但见过零件：
  "三只眼" + "绿色" + "外星" → Transformer 动态组合语义
  图片编码器提取"眼球排列""绿色纹理"
  → 余弦相似度对决 → 高度吻合 → 识别成功

不是死记硬背预存类别，是理解特征组合。
本质：文本-图像语义翻译器。
```

### 提示词工程

推理时用 `"a photo of a [描述]"` 模板 → 让推理文本分布对齐训练时的网页标题分布，显著提升准确率。

### 关键认知纠偏

| ❌ 误区 | ✅ 正确 |
|---------|--------|
| 相似度用向量卷积算 | 用**点积/余弦相似度**，绝非卷积 |
| 预存了各种类别向量 | 文本向量**实时动态计算** |
| 数据必须人工标注 | 全自动抓取，靠规模和对比学习克服噪声 |
| 只能处理图像和文本 | 思想可跨界：音频(CLAP)、视频、药物分子(DrugCLIP) |

### 跨界扩展：CLAP（音频-文本 CLIP）

```
双塔思想平移到音频：
  图像编码器 → 音频编码器（Wav2Vec 2.0）
  文本编码器 → 保持不变
  数据 → YouTube/播客的"音频-字幕"配对
  应用 → 语音指令理解、用文字搜音效（"雷雨声"）
```

### 一句总结

> CLIP 通过双塔架构和对比学习，将视觉和语言"翻译"成同一种数学语言；凭借基础语义单元的乐高式组合能力，实现零样本泛化，彻底改变了多模态 AI。
