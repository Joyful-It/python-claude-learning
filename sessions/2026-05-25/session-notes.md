## 第1题：自注意力计算流程 + 为什么缩放（背！）

### 五步流程

```
① 线性投影：  X @ W_Q = Q,   X @ W_K = K,   X @ W_V = V
              三个独立矩阵，投影到不同角色

② 算相似度：  scores = Q @ K^T
              每个词对每个词的关联度分数矩阵

③ 缩放：      scores / √d_k
              为什么？d_k 越大 → Q·K 方差越大 → softmax 输出极端 → 梯度趋零
              除以 √d_k 把方差拉回 1，保护梯度

④ Softmax：   weights = softmax(scores)
              转成概率分布，每行和=1

⑤ 加权求和：  output = weights @ V
              注意力权重 × V → 挑着看，相关的多取，无关的少取
```

### 缩放原因一句口诀

> d_k 越大点积越飘，softmax 梯度死掉。除以 √d_k，方差回 1，梯度活。

## 第2题：多头注意力 vs 单头

### 单头的问题

一个头只能学一种关系模式（比如只看到"主语-谓语"语法关系）。

### 多头的优势

```
不同头 = 不同子空间 = 不同视角

头1 → 语法关系（主语-谓语）
头2 → 语义关系（同义词）
头3 → 指代关系（"它"=上文"猫"）
头n → ……

所有头并行计算，各看各的，最后拼起来。
```

### 多头公式

```
① 投影到低维：head_i = Attention(Q@W_Qi, K@W_Ki, V@W_Vi)
               每个头维度 d_k = d_model // n_heads

② 拼接：      MultiHead = Concat(head_1, ..., head_n)

③ 线性映射：  output = MultiHead @ W_O
               拼完投影回 d_model，保持输入输出同维度
```

## 第3题：编码器 vs 解码器——掩码区别

| | 编码器 | 解码器 |
|------|------|------|
| 能看到什么 | **双向**——同时看前后 | **单向**——只看位置 i 之前 |
| 用什么掩码 | 无可选，只用 Padding Mask | **因果掩码（下三角矩阵）** |

### 为什么解码器必须用因果掩码？

```
生成时逐词输出，生成第 i 个词时：
  只能看 1, 2, ..., i-1（已生成的）
  不能看 i+1, i+2, ... （未来的，答案还没出来）

因果掩码 = 下三角矩阵：
  [1 0 0]     词1只能看词1
  [1 1 0]     词2能看词1,2
  [1 1 1]     词3能看词1,2,3

未来位置直接置 -∞ → softmax 后 → 权重为 0
```

## 第4题：输入预处理 + 位置编码

### 输入四步

```
① 分词：BPE/WordPiece → "你好世界" → [101, 203, 456]
② 词嵌入：查表 → 每个 ID → d_model 维向量
           再 × √d_model（防止嵌入方差太小被位置编码淹没）
③ 位置编码：sin/cos 或可学习 → 同样 d_model 维向量
④ 相加：   输入 = 词嵌入 + 位置编码
           为什么不拼起来而是加？维度不变，两种信息融合
```

### 位置编码解决什么问题？

```
Transformer 没有先后顺序概念——
"吃饱了没" 和 "没吃饱了" 的自注意力矩阵完全一样。

位置编码注入位置信息 → 模型知道"饱"在第 2 位还是第 4 位。
```

## 第5题：FFN（前馈网络）

### 位置 & 作用

```
位置：跟在每个注意力子层后面（每层一个 FFN）

作用：注意力是"集体开会交流信息"
      FFN 是"会后个人消化吸收"
      每个位置独立通过同一套 FFN → 非线性特征变换
```

### 典型结构

```
FFN(x) = Dropout( GELU( x@W1 + b1 ) ) @ W2 + b2
          ↑                ↑                 ↑
       d_model → d_ff   激活    d_ff → d_model

d_ff 通常 = 4 × d_model
第一层升维（4倍）→ 激活 → 第二层压回原维度

为什么先升后降？
  升维给更多空间做非线性变换，降维把提取好的特征压回来
```

## 第6题：残差连接 + LayerNorm + Pre-LN

### 残差连接

```
output = x + SubLayer(x)
         ↑   ↑
      原始输入  子层处理后的结果

为什么？
  深度网络梯度传多层会衰减。
  +x 提供旁路，梯度直接原样穿回 → 和 LSTM C_t 加法一个道理
```

### LayerNorm

```
对单个样本在特征维度做归一化：mean=0, std=1
作用：训练稳定，加速收敛，不怕输入尺度变化

LayerNorm vs BatchNorm：
  BN → 对 batch 维度归一化（NLP 不适配，序列长度不同就废了）
  LN → 对每个样本自己归一化（不受 batch/序列长度影响，Transformer 标配）
```

### Post-LN vs Pre-LN

```
Post-LN（原论文）：x + Dropout( SubLayer( LayerNorm(x) ) )
                                                    ↑
Pre-LN（现代主流）：x + Dropout( SubLayer( LayerNorm(x) ) )
                                  ↑
                  LN 放在子层前面

Pre-LN 好处：深层训练更稳定，梯度传播更平滑
现代 Transformer（GPT/BERT 后续变体）基本都用 Pre-LN
```

## 第7题：Transformer vs RNN 三大优势

| # | 优势 | 解释 |
|---|------|------|
| ① | **完全并行** | RNN 串行逐词算，Transformer 所有词同时算 → GPU 利用率拉满 |
| ② | **长程依赖直连** | RNN 梯度穿 N 步衰减，Transformer 自注意力每个词直接看全局 |
| ③ | **扩展效率** | 并行 → 训练快几十倍 → 能堆更多层、喂更多数据 → 大模型基石 |

### 之前记得的补充（05-22 课件对比表）

```
RNN 保留了 Transformer 没有的优势：
  ✅ 线性复杂度 O(n)（Transformer O(n²)）
  ✅ 天然流式处理无限长序列
  ✅ 状态连续性更适合控制类任务

结论：不是谁替代谁，是互补，常混合使用
```

## 扩展二：显卡（GPU）和大模型的关系

### CPU 为什么不行？

```
大模型 = 海量矩阵乘法  × 海量参数 × 海量数据

CPU：4-16 核 → 同时算 16 个
GPU：几千个核 → 同时算几千个

GPT-3 一次前向 ≈ 3500 亿次浮点运算
→ CPU 等到老，GPU 几秒
```

### 关键指标速查

| 指标 | 含义 | 影响什么 |
|------|------|---------|
| **显存 VRAM** | GPU 自带内存大小 | 模型能不能装进去 |
| **算力 TFLOPS** | 每秒几万亿次运算 | 训练/推理多快 |
| **带宽** | 数据进出速度 | 大 batch 卡不卡 |

### 训练 vs 推理的显存

```
推理 = 模型权重 × 1          → 显存 ≈ 模型大小
训练 = 模型 + 梯度 + 优化器状态   → 显存 ≈ 模型 × 4

例如 LLaMA-7B（14G 权重）：
  推理 → 14G 显存够用
  训练 → 需要 ~56G 显存起步
```

### 显存不够？三招

```
① 量化：fp32 → fp16/int8 → 显存减半甚至 1/4
② 梯度累积：batch_size 小，算 N 次再更新，效果等于大 batch
③ 模型并行：一张卡装不下，拆到多张卡上
```

## 扩展三：ML → DL 数学一条线

```
          全 是 wx+b 的变体
              ↓
线性回归    ŷ = wx + b，MSE 损失，令导数为 0 → 解析解
    ↓
逻辑回归    z = wx + b，ŷ = sigmoid(z)，交叉熵损失
    ↓           ↑ 多了一层 sigmoid
多层感知器  x → W1x+b1 → σ → W2x+b2 → ŷ
    ↓       ↑ 隐藏层，σ 折弯让网络能画曲线（XOR 问题一条线分不开）
深度网络    多层堆叠，每层 w,b 都通过反向传播（链式法则）更新
    ↓       ↑ ∂L/∂w₁ = ∂L/∂ŷ × ∂ŷ/∂z₂ × ∂z₂/∂a₁ × ∂a₁/∂z₁ × ∂z₁/∂w₁
CNN         把 wx+b 换成卷积：只和邻居乘，kernel 全图滑 → 共享参数
    ↓       ↑ 还是矩阵乘法，只是权重矩阵变稀疏了
RNN/LSTM    wx+b 加上时间：h_t = x_t@W_x + h_{t-1}@W_h + b
    ↓       ↑ 多了一项上一刻的自己，参数跨时刻共享
Transformer 把 W 打出三份 (W_Q,W_K,W_V)，用 Attention 替代循环
            ↑ 最底层还是 x@W，只是计算方式从顺序变成全局注意力
```

## 扩展四：梯度全家桶——概念 / 正负 / 消失 / 爆炸 / 累积

### 一、梯度是什么？

```
梯度 = 偏导数向量 = 每个参数的"最陡上坡方向"

站在山上想下山：
  梯度 → 告诉你往上爬最快的方向
  负梯度 → 往下走最快的方向（你要的！）

loss 大 = 山顶（预测差），loss 小 = 山脚（预测好）
w_new = w_old - lr × ∂L/∂w   ← 负号 = 往 loss 下降方向走
```

### 二、正梯度 vs 负梯度（两个层面）

```
层面 1 — 做决策（值和方向）：

  loss = (w - 3)² → ∂L/∂w = 2(w - 3)

  w=5: 梯度 = +4（正）→ w 增→loss 增 → w 要减小 → w -= lr×4 ✅
  w=1: 梯度 = -4（负）→ w 增→loss 减 → w 要增大 → w -= lr×(-4) ✅
  
  不管正负，往负梯度走 → loss 一定降

层面 2 — 描述训练现象：
  正梯度 = 参数把 loss 推高了（需要减小 w）
  负梯度 = 参数把 loss 拉低了（需要增大 w）
```

### 三、梯度消失

```
原因：链式连乘，每层导数都 ≤ 1

  sigmoid' 最大 0.25 × 0.25 × 0.25 × 0.25 → 乘 4 层就 ≈ 0.004
  
  传到前几层 → 梯度 ≈ 0 → 参数不动 → 废了

解法：
  ① ReLU：正区间导数恒为 1，不衰减（最常用）
  ② LSTM：C_t = f×C_{t-1} + i×~C，加法 → 梯度 C_t/C_{t-1}=f_t 直传
  ③ 残差：y = x + F(x)，+x 旁路导数 = 1
  ④ Layer/Batch Norm：稳定分布
```

### 四、梯度爆炸

```
原因：同样连乘，但 |W| > 1

  RNN 尤其严重——同一个 W_h 在时间轴反复乘自己
  1.5¹⁰⁰ → 天文数字 → NaN

解法：
  ① 梯度裁剪：clip_grad_norm_(max_norm=1)，超限就等比缩小
  ② 好初始化：Xavier/He 让 W 初始值刚好不膨胀
  ③ Layer/Batch Norm
  ④ LSTM/GRU 门控
```

### 五、消失 vs 爆炸——根本原因一样吗？

```
根本原因完全一样：链式连乘 + 雅可比矩阵的指数效应

  |特征值| < 1 → 连乘 → 0      = 消失
  |特征值| > 1 → 连乘 → ∞      = 爆炸

同根生，方向不同。LSTM 治消失靠加法结构，治爆炸靠门控（sigmoid 门上限 1）。
```

### 六、梯度累积——显存不够时的救命技巧

```
问题：batch_size=64 → 显存不够，跑不了

解法：
  batch_size=16, accumulation_steps=4

  step 1: 16 样本 → backward → 梯度存 .grad 里，不清零
  step 2: 16 样本 → backward → 梯度累加到 .grad
  step 3: 16 样本 → backward → 继续累加
  step 4: 16 样本 → backward → 
           → 梯度等效于 64 个样本 → optimizer.step() → zero_grad()
  
效果：等效 batch_size=64，但显存只用 batch_size=16 的量
代价：多算了 3 次 backward 才更新，训练时间拉长
```

```python
# 梯度累积标准写法
accumulation_steps = 4
optimizer.zero_grad()  # 开始前清零
for i, (X_batch, y_batch) in enumerate(train_loader):
    y_pred = model(X_batch)
    loss = criterion(y_pred, y_batch) / accumulation_steps  # ⚠️ 必须除以累积步数
    loss.backward()  # 梯度累加，不清零

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()       # 攒够了才更新
        optimizer.zero_grad()  # 更新后才清零
```
