## 第一节：RNN 循环神经网络（背！）

### RNN vs 全连接 vs CNN

| | 全连接 | CNN | RNN |
|------|------|------|------|
| 公式 | `X@W+b` | `input⊗kernel+b` | `h_t = tanh(x_t@W_x + h_{t-1}@W_h + b)` |
| 在哪算 | 一次 | 空间滑动 | **时间传递** |
| 参数 | 各层各的 | 卷积核共享 | **W_x, W_h 跨时刻共享** |
| 激活 | ReLU | ReLU | **tanh**（防数值爆炸） |
| 谁教 | 反向传播 | 反向传播 | **BPTT**（反向传播穿时间） |

### RNN 核心——比全连接只多一项

```
全连接：  ŷ = x @ W + b
RNN：    h_t = tanh(x_t @ W_x + h_{t-1} @ W_h + b)
                    ↑ 新输入          ↑ 上一刻记忆

多出来的 h_{t-1}@W_h = 上一刻自己算的东西，作为额外输入
```

### 推一个例子

```
"我 爱 你" → h₀ = [0,0]

"我"：  h₁ = tanh("我"@W_x + h₀@W_h + b) = [0.3, 0.1]    ← 记住了"我"
"爱"：  h₂ = tanh("爱"@W_x + h₁@W_h + b) = [0.8, 0.5]    ← 记住了"我爱"
"你"：  h₃ = tanh("你"@W_x + h₂@W_h + b) = [0.9, 0.7]    ← 记住了"我爱你"

最后 h₃ 拿去分类 → 包含了整句信息
```

### RNN 的致命问题

```
句子越长 → h 越压缩 → 开头信息被覆盖 → 梯度消失
"我昨天去了那家新开的在王府井旁边的川菜馆吃饭"
  → 传到"吃饭"时，"昨天"的信息早没了
```

## 第二节：LSTM 长短期记忆网络（背！）

### RNN→LSTM：加了一条"传送带"

```
RNN：  只传 h（压缩记忆，容易覆盖）

LSTM： 传两个状态
         C_t = 长期记忆（笔记本，内部传送带，不被过滤）
         h_t = 短期记忆（对外输出，被输出门过滤）
```

### 三门公式（结构全一样，W和b各不同）

```
遗忘门：f = sigmoid(W_f·[h, x] + b_f)    → 哪个旧信息该忘？（0~1）
输入门：i = sigmoid(W_i·[h, x] + b_i)    → 哪个新信息该记？（0~1）
输出门：o = sigmoid(W_o·[h, x] + b_o)    → 当前该对外说什么？（0~1）

新候选：～C = tanh(W_c·[h, x] + b_c)     → 产生新信息（-1~1）
```

### 一个时刻的完整执行顺序

```
① 三个门同时拿 [h, x] → 并行算出 f, i, o（各出0~1）
② C_t = f × C_{t-1} + i × ～C           ← 遗忘旧 + 写入新
③ h_t = o × tanh(C_t)                    ← 从长期记忆里挑着说
④ C_t, h_t 分别传给下一时刻
```

### 三门分工口诀

```
遗忘门："这个，你走"    → 擦旧笔记
输入门："这个，你进"    → 写新笔记
输出门："这个，你说"    → 对外输出
```

### 为什么门用 sigmoid，候选用 tanh？

```
sigmoid → 0~1 → 阀门：0=关死，1=全开，天生"控制"
tanh    → -1~1 → 信息：能正能负，零中心化，表达丰富
```

### 梯度怎么教三门？

```
"火锅"进来 → 遗忘门忘了→预测崩→梯度罚遗忘门W_f→下次别忘
"火锅"进来 → 输入门没记→预测崩→梯度罚输入门W_i→下次多记
"火锅"进来 → 输出门关了→下一步缺主语→梯度罚输出门W_o→下次开门

万次训练后：遗忘门学会见到核心名词前忘旧，输入门学会记关键词，输出门学会说重要信息。
```

### C_t 和 h_t 分工

```
C_t = 笔记本（内部专线，全量保留，不被输出门过滤）  ← 长期
h_t = 对外说话（被输出门过滤过，不是全部知识）      ← 短期
```

### 参数共享 + 梯度累加

```
W_x、W_h 在所有时刻是同一套参数（和CNN卷积核全图共享一样）
反向传播：W_f 在 6 个时刻各被改一次 → 6 次梯度累加 → 一次更新
```

## 第三节：GRU vs LSTM vs RNN 三兄弟

| | RNN | LSTM | GRU |
|------|------|------|------|
| 状态 | 1(h) | 2(C, h) | 1(h) |
| 门 | 0 | 3(遗忘+输入+输出) | 2(重置+更新) |
| 参数 | 最少 | 最多(4×RNN) | 中等(3×RNN) |
| 长句 | ❌ | ✅ | ✅(接近LSTM) |
| 速度 | 最快 | 慢 | 中 |

```
GRU = LSTM 简化版：
  重置门 ≈ 遗忘门（忘多少旧h）
  更新门 ≈ 遗忘门+输入门合一（新旧各占多少比例）
  h_t = (1-z)×h_{t-1} + z×新候选
```

## 第五节：新增错题本

| 问 | 模糊 | 正确 |
|---|------|------|
| y 要不要 copy？ | 不确定 | **不用**——y 只读不写，没赋值操作 |
| 三门谁先谁后？ | 有先后 | **同时算**三门→但 C_t 等遗忘+输入→h_t 等 C_t |
| C_t h_t 哪个长期？ | C_t 是短的？ | **C_t 长期**（内部传送带不被过滤），h_t 短期（被输出门过滤） |
| LSTM 不用 tanh？ | 对 | **门用 sigmoid**(0~1控制)，**候选用 tanh**(-1~1信息) |
| RNN 激活为什么 tanh？ | 和全连接一样 ReLU | 循环用 ReLU 值会**爆炸**，tanh 有界 -1~1 |
| W_x 和 W 全连接一样？ | 每时刻参数不同 | **跨时刻共享同一套**，和 CNN 卷积核全图滑一样 |
