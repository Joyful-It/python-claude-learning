# LoRA 低秩微调

## 1. 参数是什么？

深度学习里的参数，指的是模型内部可以通过训练自动学习出来的数值。

常见参数包括：

| 参数类型 | 例子 |
|---|---|
| 线性层权重 | `nn.Linear` 中的 `weight` |
| 偏置 | `bias` |
| 卷积核 | CNN 中的卷积核权重 |
| Embedding | 大语言模型中的词向量表 |
| Attention 权重 | Q、K、V、O 投影矩阵 |
| LayerNorm 参数 | 缩放和平移参数 |

下面这些不是模型参数：

| 名称 | 是否是模型参数 |
|---|---|
| 训练数据 | 不是 |
| batch size | 不是 |
| epoch | 不是 |
| 学习率 | 不是，是超参数 |
| loss | 不是，是损失值 |
| 梯度 | 不是，是更新方向 |

平时说的大模型规模，通常就是指：

模型参数量

例如：

| 模型规模 | 含义       |
| -------- | ---------- |
| 0.5B     | 5 亿参数   |
| 1.5B     | 15 亿参数  |
| 7B       | 70 亿参数  |
| 13B      | 130 亿参数 |
| 70B      | 700 亿参数 |

这里的 `B` 是 `Billion`，表示十亿。

所以一个 `7B` 模型，就是大约有 **70 亿个参数**。

## 2. 全参数微调

全参数微调是用新数据继续训练整个模型，所有权重都参与更新。

假设模型中有一个线性层：

```text
y = Wx
```

其中：

- `x` 是输入；
- `W` 是原模型中的权重矩阵；
- `y` 是输出。

全参数微调时，会直接更新 `W`。

训练后可以理解成：

```text
W' = W + ΔW
```

其中：

| 符号 | 含义                       |
| ---- | -------------------------- |
| `W`  | 原始预训练权重             |
| `ΔW` | 微调过程中学到的权重变化量 |
| `W'` | 微调后的新权重             |

例如：

```text
W: 4096 × 4096
ΔW: 4096 × 4096
```

因为它们要相加，所以形状必须一致。

全参数微调的问题是：

1. 训练参数量大；
2. 显存消耗高；
3. 训练成本高；
4. 每个任务都要保存一份完整模型；
5. 数据少时容易破坏原模型能力，也就是灾难性遗忘。

## 3. LoRA微调是什么

**LoRA 是一种参数高效微调方法。**

它的核心思想是：

冻结原来已经训练好的大模型参数，不直接修改原模型的大权重矩阵，而是在部分线性层旁边新增两个很小的可训练矩阵，只训练这两个小矩阵，让模型适应新任务。

LoRA 是大模型时代的一种参数高效迁移学习方法

迁移学习的核心思想是：

把一个模型在大规模数据上学到的通用能力，迁移到一个新的任务上。

例如：

- 原模型：已经学会通用语言理解能力；
- 新任务：法律问答、医学问答、金融分析、代码生成、食物热量问答；
- 微调：让原模型更适合这个新任务。

LoRA 不是从零训练模型，而是在已有预训练模型基础上，用少量参数完成任务适配，所以属于迁移学习中的一种微调方式。

---

## 4. LoRA 的核心公式

全参数微调可以写成：

```text
y = Wx + ΔWx
```

LoRA 不直接训练完整的 `ΔW`，而是用两个小矩阵相乘表示它：

```text
ΔW = B × A
```

加入 LoRA 后变成：

```text
y = Wx + BAx
```

实际中还会加一个缩放系数：

```text
y = Wx + (α / r)BAx
```

LoRA不是：

```text
先得到完整 ΔW，再分解成 A 和 B
```

而是：

```text
一开始就不训练完整 ΔW
直接规定 ΔW = BA
然后通过训练学习 A 和 B
```

其中：

| 符号 | 含义 |
|---|---|
| `W` | 原模型权重，冻结不训练 |
| `A` | LoRA 新增的小矩阵 |
| `B` | LoRA 新增的小矩阵 |
| `r` | 秩，控制 LoRA 中间维度 |
| `α` | 缩放系数，控制 LoRA 分支影响强度 |

---

## 5. LoRA 为什么能省参数？

假设原权重矩阵是：

```text
W: 4096 × 4096
```

如果全参数微调，训练完整的 `ΔW`，参数量是：

```text
4096 × 4096 = 16,777,216
```

大约是 **1677 万个参数**。

如果使用 LoRA，设：

```text
r = 8
```

那么两个小矩阵是：

```text
A: 8 × 4096
B: 4096 × 8
```

LoRA 需要训练的参数量是：

```text
8 × 4096 + 4096 × 8 = 65,536
```

大约是 **6.5 万个参数**。

对比：

| 方法 | 可训练参数量 |
|---|---:|
| 全参数微调 | 16,777,216 |
| LoRA 微调，r=8 | 65,536 |

所以 LoRA 的优势不是最终矩阵形状变小，而是：

> **真正需要训练和保存的参数变少了。**

## 6. LoRA 的微调过程

LoRA 之所以叫微调，是因为它仍然基于预训练模型，用新任务数据进行继续训练和任务适配；但它不是更新全部模型参数，而是冻结原模型，只训练新增的低秩适配参数。

区别在于：

```text
全参数微调：更新原模型的 W
LoRA 微调：冻结原模型的 W，只更新新加的 A 和 B
```

训练过程如下：

```text
新任务数据输入模型
↓
原模型 W 计算原始输出
↓
LoRA 分支 A、B 计算修正输出
↓
两部分相加得到最终输出
↓
和真实答案计算 loss
↓
反向传播
↓
只更新 A 和 B
↓
W 保持不变
```

所以 LoRA 的微调过程就在：

> **A 和 B 从初始值变成适合新任务的值。**

## 8. 为什么叫“低秩”？

因为：

```text
ΔW = B × A
```

其中：

```text
B: 输出维度 × r
A: r × 输入维度
```

两个矩阵相乘后，`ΔW` 的秩最多不会超过 `r`。

也就是：

```text
rank(ΔW) ≤ r
```

如果：

```text
r = 8
```

那么 `ΔW` 最多就是秩 8。

所以叫：

> **低秩微调。**

---

## 9. A 和 B 是怎么确定的？

A 和 B 的**尺寸**由原权重矩阵和秩 `r` 决定。

如果：

```text
W: 输出维度 × 输入维度
```

那么：

```text
A: r × 输入维度
B: 输出维度 × r
```

例如：

```text
W: 4096 × 4096
r = 8
```

那么：

```text
A: 8 × 4096
B: 4096 × 8
```

但是 A 和 B 里面的具体数值不是人工指定的，而是通过训练自动学习出来的。

通常初始化方式是：

```text
A 随机初始化
B 初始化为 0
```

这样刚开始时：

```text
BA = 0
```

所以模型一开始基本等价于原模型，不会突然破坏原模型能力。

随着训练进行，A 和 B 被不断更新，LoRA 分支逐渐学到新任务需要的修正能力。

## 10. LoRA 在代码中的核心逻辑

一个普通线性层原来是：

```python
base_output = x @ W.T
```

加入 LoRA 后变成：

```python
base_output = x @ W.T
lora_output = x @ A.T @ B.T * scaling
output = base_output + lora_output
```

也可以写成 PyTorch 模块形式：

```python
class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=8, lora_alpha=16):
        super().__init__()

        self.base_layer = nn.Linear(in_features, out_features)

        for param in self.base_layer.parameters():
            param.requires_grad = False

        self.lora_A = nn.Linear(in_features, r, bias=False)
        self.lora_B = nn.Linear(r, out_features, bias=False)

        self.scaling = lora_alpha / r

        nn.init.zeros_(self.lora_B.weight)

    def forward(self, x):
        base_output = self.base_layer(x)
        lora_output = self.lora_B(self.lora_A(x)) * self.scaling
        return base_output + lora_output
```

核心只有三行：

```python
base_output = self.base_layer(x)
lora_output = self.lora_B(self.lora_A(x)) * self.scaling
output = base_output + lora_output
```

## 11. LoRA 的优点

| 优点 | 解释 |
|---|---|
| 省显存 | 原模型参数冻结，不需要为所有参数保存梯度 |
| 训练快 | 只训练少量 LoRA 参数 |
| 保存小 | 每个任务只保存 LoRA 权重 |
| 任务切换方便 | 同一个基础模型可以加载不同 LoRA |
| 保留原模型能力 | 原模型主体参数不被破坏 |

---

## 12. LoRA 的局限性

LoRA 不是万能的，它也有缺点：

1. 如果基础模型能力很弱，LoRA 很难让它突然变强；
2. 如果新任务和原模型差异太大，只训练少量参数可能不够；
3. `r` 需要实验选择，`r` 越大，表达能力越强，但参数量更多，也可能过拟合。
4. 加在哪些层也需要根据任务调整；
5. 如果任务非常复杂，LoRA 效果可能不如全参数微调。
