# 零样本学习：OpenAI CLIP

> 本文从零开始讲解 CLIP 模型的原理与实战，带你理解 **"为什么 AI 能在没有见过某个类别的情况下，依然能识别它"**。
---

## 🌐 一、什么是零样本学习（Zero-Shot Learning）

### 1. 传统 CV 的困境：闭集测试

传统的图像分类模型（如 ResNet、YOLO）本质上都是 **"填鸭式"的监督学习**。

> **打个比方：** 一个模型在训练时只见过"猫"和"狗"的照片。当它面对一张"汽车"照片时，模型**只能从"猫"和"狗"里硬选一个**，结果必然是灾难性的误判。

这种模式的痛点：
- 需要耗费巨额人力进行图片标注
- 分类标签被死死固定在代码里，无法灵活扩展

这种模式被称为 **"闭集（Closed-set）"** 控制。

---

### 2. 零样本学习的破局：开集泛化

**零样本学习（Zero-Shot Learning）** 的核心定义是：

> **模型在完全不接触某一类别的训练样本（图片）的情况下，直接能够对该类别进行准确的分类和识别。**

#### 人类的类比

你从未见过"麒麟"，但如果我告诉你"麒麟长着狮子的头、鹿的角、牛的尾巴、身上满是鳞片"，当一幅麒麟的画作摆在面前时，你一定能认出来。

**关键洞察：** 人类可以通过 **语言（文本语义）** 作为桥梁来实现零样本识别。

#### AI 的实现

让计算机不再孤立地看图片，而是将 **视觉特征** 与 **文本语义（自然语言）** 进行跨模态对齐。

---

## ⚡ 二、CLIP 的训练机制：双塔架构与对比学习

CLIP 的核心思想是放弃传统的"人工预定义离散标签"的监督范式，转而从海量 **图文对数据** 中直接学习跨模态的特征对齐。

### 2. 训练全流程的四大阶段

#### 阶段一：多模态双塔前向传播

在含有 **N 个图文对**的 Batch 中：
- **图像** 送入 Image Encoder（如 ViT）提取视觉特征
- **文本** 送入 Text Encoder（如 Transformer）提取文本特征
- 两者经过线性投影层，映射到 **相同的共享嵌入空间**，特征维度为 **D**

#### 阶段二：特征 L2 归一化

为了消除特征尺度的影响，在计算相似度前，对所有特征向量执行 **L2 归一化**。

> 归一化后，向量之间的**点积运算**在数学上完全等价于它们的**余弦相似度**。

#### 阶段三：合成 N×N 相似度方阵

利用矩阵乘法 `Logits = A × Bᵀ` 批量计算所有图文对的相似度：

|          | 文本₁ | 文本₂ | ... | 文本ₙ |
|----------|:-----:|:-----:|:---:|:-----:|
| **图像₁** | ✅ 正样本 | ❌ 负样本 | ... | ❌ 负样本 |
| **图像₂** | ❌ 负样本 | ✅ 正样本 | ... | ❌ 负样本 |
| **...**   | ...   | ...   | ... | ...   |
| **图像ₙ** | ❌ 负样本 | ❌ 负样本 | ... | ✅ 正样本 |

- **对角线** → 正确的正样本配对（Positive Pairs）
- **非对角线** → 不匹配的负样本干扰项（Negative Pairs）

#### 阶段四：双向 InfoNCE 损失

CLIP 同时从两个方向计算损失：

1. **图像到文本损失**：以图像为基准，计算正确文本在所有候选文本中的概率
2. **文本到图像损失**：以文本为基准，计算正确图片在所有候选图片中的概率

最终效果：

> **拉近（Pull）** 正样本对的特征距离，**推开（Push）** 负样本对的特征距离。

---

### 3. 极简数值推导（N=2, D=2）

为了让上述数学过程更直观，我们用一个 **最简单的例子** 来演示。

**设定：**
- 批次大小 N = 2（2个图文对）
- 特征维度 D = 2
- 温度参数 τ = 0.5

---

#### 步骤一：特征提取与归一化

假设经过编码器和 L2 归一化后的特征矩阵为：

```
视觉矩阵 A：
┌             ┐
│ 1.0   0.0  │  ← 图像₁ 的特征向量 V₁
│ 0.0   1.0  │  ← 图像₂ 的特征向量 V₂
└             ┘

文本矩阵 B：
┌                 ┐
│ 0.707  0.707   │  ← 文本₁ 的特征向量 T₁
│ 0.0    1.0     │  ← 文本₂ 的特征向量 T₂
└                 ┘
```

---

#### 步骤二：计算相似度矩阵

通过矩阵乘法计算两两相似度（余弦相似度）：

```
Logits = A × Bᵀ

    ┌ 1.0  0.0 ┐   ┌ 0.707  0.0 ┐   ┌ 0.707  0.0 ┐
    │ 0.0  1.0 │ × │ 0.707  1.0 │ = │ 0.707  1.0 │
    └          ┘   └            ┘   └            ┘

          A               Bᵢ         相似度矩阵
```

**结果分析：**
- 位置 (1,1)：得分 **0.707** ✅ 图像₁ ↔ 文本₁（正样本）
- 位置 (2,2)：得分 **1.000** ✅ 图像₂ ↔ 文本₂（正样本）
- 其余位置：❌ 负样本干扰项

---

#### 步骤三：温度缩放 + Softmax 概率

将得分除以温度系数 τ=0.5（放大差异），再计算 Softmax：

```
指数化后的矩阵：
┌                        ┐
│ e¹·⁴¹⁴    e⁰          │   ≈ ┌ 4.112  1.0   ┐
│ e¹·⁴¹⁴    e²          │     │ 4.112  7.389 │
└                        ┘     └              ┘
```

**行 Softmax（图像视角 → 预测文本）：**

| 图像  |            预测文本₁ 的概率            |            预测文本₂ 的概率            |
| :-: | :-----------------------------: | :-----------------------------: |
| 图像₁ |  4.112/(4.112+1.0) = **0.804**  |     1.0/(4.112+1.0) = 0.196     |
| 图像₂ | 4.112/(4.112+7.389) = **0.358** | 7.389/(4.112+7.389) = **0.642** |

**列 Softmax（文本视角 → 预测图像）：**

| 文本  |            预测图像₁ 的概率            |           预测图像₂ 的概率           |
| :-: | :-----------------------------: | :---------------------------: |
| 文本₁ | 4.112/(4.112+4.112) = **0.500** |  4.112/(4.112+4.112) = 0.500  |
| 文本₂ |     1.0/(1.0+7.389) = 0.119     | 7.389/(1.0+7.389) = **0.881** |

---

#### 步骤四：计算总损失
![[Pasted image 20260522100717.png]]

**图像到文本损失：**
```
L_i2t = -(ln(0.804) + ln(0.642)) / 2 = -(-0.218 - 0.443) / 2 ≈ 0.331
```

**文本到图像损失：**
```
L_t2i = -(ln(0.500) + ln(0.881)) / 2 = -(-0.693 - 0.127) / 2 ≈ 0.410
```

**总损失：**
```
L_total = (0.331 + 0.410) / 2 = 0.371
```

---

#### 步骤五：梯度更新与收敛验证

损失函数对相似度矩阵的梯度：

```
∇ = ┌ -0.348   0.158  ┐
    └  0.158  -0.238  ┘
```
![[Pasted image 20260522101713.png|438]]

- **对角线负数** → 模型会**增大**正样本相似度（Pull 拉近）
- **非对角线正数** → 模型会**减小**负样本相似度（Push 推开）

**更新文本₁的特征向量（学习率 η=0.1）：** $$ \begin{aligned} T_{1\text{\_new\_raw}} &= [0.707, 0.707] - 0.1 \times [-0.348, 0.158] \\ &= [0.742, 0.691] \end{aligned} $$ **💡 补充步骤L2 归一化（将参数更新后的向量长度重新校准为 1.0）**
1. 计算更新后向量的模长（长度）： $$\|T_{1\text{\_new\_raw}}\| = \sqrt{0.742^2 + 0.691^2} = \sqrt{1.0280} \approx 1.0139$$ 2. 执行归一化，得到标准的特征向量 $T_{1\text{\_new}}$： $$T_{1\text{\_new}} = \left[ \frac{0.742}{1.0139}, \frac{0.691}{1.0139} \right] = [0.732, 0.682]$$ --- **验证： 更新后的标准 $T_1$ 与正样本 $V_1 [1.0, 0.0]$ 的余弦相似度：** $$ \text{sim\_new} = 0.732 \times 1.0 + 0.682 \times 0.0 = 0.732 $$ **结果：** 严格归一化后，标准的余弦相似度从 **0.707** 提升至 **0.732** 模型正在向正确方向收敛！

## 💻 三、实战：零样本预测系统

下面展示如何用 **不到 20 行核心代码**，实现一个零样本图片分类器。

### 1. 安装依赖

```bash
pip install torch transformers pillow
```

### 2. 完整代码（clip_demo.py）

```python
import torch
from PIL import Image
from transformers import CLIPProcessor, CLIPModel

# -------------------- 1. 加载模型 --------------------
model_id = "openai/clip-vit-base-patch32"
model = CLIPModel.from_pretrained(model_id)
processor = CLIPProcessor.from_pretrained(model_id)
print("--- CLIP 模型加载成功！ ---")

# -------------------- 2. 读取图片 --------------------
image_path = "to_be_checked/1.jpg"  # 请替换为你的图片路径
image = Image.open(image_path)

# -------------------- 3. 动态定义文本标签 --------------------
labels = [
    "a photo of a cat",
    "a photo of a dog",
    "a photo of a luxury car",
    "a forest in the autumn",
    "a handsome boy",
    "a normal boy"
]

# -------------------- 4. 预处理（图像+文本） --------------------
inputs = processor(text=labels, images=image, return_tensors="pt", padding=True)

# -------------------- 5. 推理（关闭梯度计算） --------------------
with torch.no_grad():
    outputs = model(**inputs)

# -------------------- 6. 计算概率分布 --------------------
logits_per_image = outputs.logits_per_image   # 原始相似度得分
probs = logits_per_image.softmax(dim=-1)      # Softmax → 概率（总和100%）

# -------------------- 7. 打印结果 --------------------
print("\n--- 零样本识别（Zero-Shot）预测结果 ---")
for label, prob in zip(labels, probs[0]):
    print(f"文本描述: '{label}' ----> 匹配概率: {prob.item() * 100:.2f}%")

# -------------------- 8. 输出最终决策 --------------------
max_index = torch.argmax(probs).item()
print(f"\n💡 AI 最终认为：这张图片描述的是 -> '{labels[max_index]}'")
```

### 3. 代码逻辑拆解

| 步骤  | 做了什么               | 核心概念                        |
| :-: | :----------------- | :-------------------------- |
|  ①  | 加载预训练的 CLIP 模型和处理器 | 模型已在大规模图文对上预训练好             |
|  ②  | 读取待测试图片            | 零样本 = 模型从未在训练中见过这张图         |
|  ③  | 用自然语言定义多个候选标签      | 标签可以随意修改，不需要重新训练模型          |
|  ④  | 将图片和文本统一处理为张量格式    | CLIPProcessor 自动完成图像缩放、文本分词 |
|  ⑤  | 模型前向推理             | 计算图片特征与所有文本特征的相似度           |
|  ⑥  | Softmax 归一化为概率     | 所有标签的匹配概率之和 = 100%          |
|  ⑦  | 输出每个标签的匹配概率        | 直观看到模型"认为"图片像什么             |
|  ⑧  | 取概率最高的标签作为最终结果     | 这就是零样本分类的最终答案               |

## 📦 四、CLIP 模型选型指南

在工业界落地时，大模型的选择往往是一场关于 **体积、速度和清晰度** 的权衡。

| 模型 ID                                 |   视觉主干    |       体积        | 特点与适用场景                                                  |
| :------------------------------------ | :-------: | :-------------: | :------------------------------------------------------- |
| **`openai/clip-vit-base-patch32`**    |    ViT    | **~570 MB** 极轻量 | **性价比之王。** 将图像切分为 32x32 像素的粗方块。对硬件零压力，普通笔记本秒跑，适合高并发初筛场景。 |
| **`openai/clip-vit-large-patch14`**   |    ViT    |   ~1.7 GB 中等    | **高清平替版。** 切片细到 14x14，能识别微小细节和图片中的印刷文字。以文搜图、长文本图文检索的主力军。 |
| **`IDEA-CCNL/Chinese-CLIP-Vit-B-32`** | ViT（中文重训） |      多种体积       | **解决"不认中文"痛点。** 完美理解"汉服、二次元、外卖"等中国本土网络流行语义。              |

## 📝 总结

| 核心要点 | 说明 |
|:---------|:------|
| **什么是零样本学习** | 模型能在没见过某类图片的情况下，依然识别出它 |
| **CLIP 如何实现** | 通过双塔架构 + 对比学习，对齐视觉特征和文本语义 |
| **训练目标** | 拉近配对的图文特征（正样本），推开不匹配的（负样本） |
| **核心优势** | 无需重新训练，只要修改文本标签就能适配新任务 |
| **落地选择** | base-patch32 轻量快速，large-patch14 高清精准 |
